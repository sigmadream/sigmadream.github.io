## Python

- [Building a RAG System With Claude, PostgreSQL & Python on AWS](https://www.timescale.com/blog/building-a-rag-system-with-claude-postgresql-python-on-aws)

  - 이 기사는 Claude와 PostgreSQL을 사용하여 RAG(Retrieval-Augmented Generation) 시스템을 구축하는 방법을 설명합니다. AWS를 활용하여 데이터베이스와 AI 모델을 통합하고, 이를 통해 효율적인 정보 검색 및 생성 시스템을 만드는 과정을 안내합니다.

- [uv overtakes Poetry](https://wagtail.org/blog/uv-overtakes-poetry/)

  - 이 기사는 Python의 패키지 관리 도구인 uv가 Poetry를 대체하는 과정을 다룹니다. uv는 더 빠르고 간편한 패키지 관리 기능을 제공하며, 특히 데이터 과학 및 머신러닝 분야에서 인기를 끌고 있습니다. 이 글에서는 uv의 주요 기능과 장점, 그리고 Poetry와의 차별점을 설명합니다.

- [Understanding Numpy's einsum](https://eli.thegreenplace.net/2025/understanding-numpys-einsum/)
  - 이 기사는 NumPy의 einsum 함수에 대한 심층적인 설명을 제공합니다. einsum은 다차원 배열 간의 연산을 효율적으로 수행할 수 있는 강력한 도구로, 이 글에서는 einsum의 사용법과 다양한 예제를 통해 그 기능을 설명합니다.

## JavaScript

- [SSR Deep Dive for React Developers](https://www.developerway.com/posts/ssr-deep-dive-for-react-developers)

  - 이 기사는 React 개발자를 위한 서버 측 렌더링(SSR)에 대한 심층적인 설명을 제공합니다. SSR의 개념, 장점 및 단점, 그리고 React에서 SSR을 구현하는 방법에 대해 다룹니다. 이 글은 SSR을 통해 성능과 SEO를 개선하고자 하는 개발자들에게 유용한 정보를 제공합니다.

- [Tests are dead. Meticulous AI ‍is here.](https://www.meticulous.ai/)

  - 이 기사는 Meticulous AI라는 새로운 테스트 도구를 소개합니다. 이 도구는 기존의 테스트 방법론을 대체하며, AI를 활용하여 자동화된 테스트를 수행합니다. Meticulous AI는 코드 변경 사항을 실시간으로 감지하고, 이를 기반으로 테스트를 생성 및 실행하여 개발자들이 더 빠르고 효율적으로 코드를 검증할 수 있도록 돕습니다.

- [Common React libraries architecture](https://www.felgus.dev/blog/common-react-lib-architecture)

  - 이 기사는 React 라이브러리의 일반적인 아키텍처를 설명합니다. React 라이브러리를 설계할 때 고려해야 할 주요 요소들(예: 상태 관리, 라우팅, 스타일링 등)을 다루며, 이를 통해 개발자들이 더 나은 라이브러리를 구축할 수 있도록 돕습니다.

- [A 10x Faster TypeScript](https://devblogs.microsoft.com/typescript/typescript-native-port/)

  - 이 기사는 TypeScript의 새로운 네이티브 포트에 대한 내용을 다룹니다. 이 포트는 TypeScript의 성능을 10배 향상시키며, 더 빠른 컴파일 속도와 더 나은 개발 경험을 제공합니다. 이 글에서는 새로운 포트의 주요 기능과 이를 활용하는 방법을 설명합니다.

- [LIVE: Anders Hejlsberg on TypeScript’s Go Port](https://www.youtube.com/watch?v=NrEW7F2WCNA)

## OOP

- [The key new features in .NET 10](https://www.infoworld.com/article/3839444/the-key-new-features-in-net-10.html)

  - 이 기사는 .NET 10의 주요 새로운 기능과 개선 사항을 소개합니다. .NET 10은 성능 향상, 새로운 API 추가 및 언어 기능 개선을 포함하여 개발자들이 더 효율적으로 코드를 작성할 수 있도록 돕습니다. 이 글은 .NET 10의 주요 변경 사항과 이를 활용하는 방법을 설명합니다.

- [New HybridCache in .NET9](https://mareks-082.medium.com/new-hybridcache-in-net9-29b1fa5a674f)

  - 이 기사는 .NET 9에서 도입된 HybridCache에 대한 내용을 다룹니다. HybridCache는 메모리와 디스크 캐시를 결합하여 성능을 향상시키고, 데이터 접근 속도를 개선하는 기능을 제공합니다. 이 글에서는 HybridCache의 작동 원리와 사용 방법을 설명합니다.

- [.NET Heisenbug Mystery Theater: How Did an Exception Escape its Catch Block?](https://petabridge.com/blog/dotnet-heisenbug-mystery-theater-catch-block/)
  - 이 기사는 .NET에서 발생한 Heisenbug(디버깅 중에 사라지는 버그)에 대한 사례를 다룹니다. 이 글에서는 버그의 원인과 이를 해결하기 위한 접근 방식을 설명하며, 개발자들이 유사한 문제를 해결하는 데 도움이 될 수 있는 통찰력을 제공합니다.
